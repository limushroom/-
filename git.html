GitHub仓库创建指南

创建GitHub仓库以实现自动部署到Netlify等平台

创建GitHub仓库

按照以下步骤创建GitHub仓库，为自动部署做好准备：

1

登录GitHub

访问 github.com 并登录您的账户

2

创建新仓库

点击右上角的"+"图标，选择"New repository"

3

填写仓库信息

输入仓库名称、描述，选择公开或私有

4

初始化仓库

建议添加README文件，选择适合的.gitignore和许可证

5

创建仓库

点击"Create repository"完成创建

上传代码到GitHub仓库

您可以通过以下两种方式将代码上传到GitHub仓库：

方法一：通过网页界面上传

适合小型项目或单个文件

# 在仓库页面点击"Add file" -> "Upload files"
# 拖拽文件或选择文件上传
# 添加提交信息后点击"Commit changes"

方法二：使用Git命令行（推荐）

适合任何规模的项目，提供完整版本控制功能

# 初始化本地Git仓库（如果尚未初始化）
git init

# 添加所有文件到暂存区
git add .

# 提交更改
git commit -m "初始提交"

# 添加远程仓库地址（将USERNAME和REPO替换为您的信息）
git remote add origin https://github.com/USERNAME/REPO.git

# 推送到GitHub
git branch -M main
git push -u origin main

提示： 如果您还没有安装Git，请先访问 git-scm.com 下载并安装Git。

连接GitHub到Netlify实现自动部署

完成GitHub仓库创建后，按照以下步骤连接Netlify：

1

登录Netlify

访问 app.netlify.com 并登录

2

开始新项目

点击"Add new site" -> "Import an existing project"

3

连接GitHub

选择GitHub作为Git提供商并授权访问

4

选择仓库

从列表中选择您刚创建的GitHub仓库

5

配置构建设置

对于静态网站，通常使用默认设置即可

6

部署网站

点击"Deploy site"完成部署

完成！ 现在，每当您向GitHub仓库推送更改时，Netlify会自动重新构建和部署您的网站。
